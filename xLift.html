xLift

Value

Returns a unary function of x.

Description

Compose a function with the outputs of other functions.

Details

xLift takes a function that works on some type of value, and makes that function work on functions of those values.

xLift is a fairly abstract operator, best explained by looking at examples. The most obvious use for xLift is to take normally binary functions like +, and max that take numbers, and 'lift' it to work on functions of numbers.

A function that multiplies a number can be defined in a function that adds the output of double that number and triple that number.

double <- x := 2*x triple <- x := 3*x

sextuple <- x := double(x) + triple(x) sextuple <- x := '+'(double(x), triple(x))

This can be viewed as 'adding' the double and triple function, or composing them with addition.

Similarily, a function to check if a value is a complex number or an integer can be defined by

is_positive <- x := x > 0 is_whole <- x := round(x) == x

is_positive_and_whole <- x := is_positive(x) && is_whole(x) is_positive_and_whole <- x := '&&'(is_positive(x), is_whole(x))

sextuple and is_positive_and_whole share a common structure. They both have take one value, and call a binary function (plus, and) with the ouput of two other functions.

xLift factors out this pattern, for binary functions and higher arity functions.

sextuple <- xLift...('+', double, triple) is_positive_and_whole <- xLift...('&&', is_positive, is_whole)

Two partially applied forms of xLift are included in arrow, which are useful for cutting down on the amount of anonymous functions you need to pass to higher-order functions.

xSelect(is_positive %and% is_whole, seq(1, 3, by = 0.1)) list(1, 2, 3)
