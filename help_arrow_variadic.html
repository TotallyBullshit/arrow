Usage - Variadic

Description

Good function naming conventions can make a library easier to use. The fantastic plyr library uses a two-letter prefix to denote the input and output types of each ply function, which makes it trivial to find the correct variant of ply function. Arrow uses a few different naming conventions, though most don't need to be remembered.

Details

Function Names

Arrow functions are always prefixed with the letter 'x', followed by an uppercase letter. Method names are camel case.

xMap

xFold

xProdSetOf

Functions often come in pairs of related functions;

xExplode

xImplode

xSelect

xReject

Certain functions process collections in a certain direction; as is convention in Haskell these have a 'l' (left) or 'r' (right) suffix.

xFoldl

xFoldr

Most arrow functions have a normal version and a ellipsis (...) version; the ellipsis form has a three-dot suffix.

xMap

xMap...

Parametre Names

Arrow has a small number of parametre names, which refer to some property of the expected argument.

  • bool: a single boolean value.
  • bools: a collection of boolean values.
  • coll: a collection.
  • ims: a collection of complex values.
  • colls: a collection of collections.
  • fn: a collection of collections.
  • num: a single number.
  • nums: a collection of numbers.
  • pred: a function that returns a logical value.
  • rexp: a string to be used as a regular expression.
  • str: a string.
  • strs: a collection of strings.
  • sym: a symbol or string.
  • val: any R value.
